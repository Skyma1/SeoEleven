# Данные кейсов

Этот файл содержит структуру данных для кейсов, которая используется на странице `/cases`.

## Текущая реализация

Данные хранятся в файле `casesData.js` и загружаются напрямую в компонент `CasesPage`.

## Интеграция с админкой/API

Для интеграции с админкой или API необходимо:

### 1. API Endpoint

Создать эндпоинт для получения кейсов:

```
GET /api/cases
```

### 2. Структура ответа API

API должен возвращать данные в следующем формате:

```json
{
  "cases": [
    {
      "id": 1,
      "title": "Заголовок кейса",
      "client": "Название клиента/компании",
      "description": "Описание проекта",
      "category": "SEO | GEO / AI SEO | Разработка | Автоматизация",
      "period": "Период работы (например, '6 месяцев')",
      "results": [
        {
          "label": "Название метрики",
          "value": "Значение (например, '+250%')"
        }
      ],
      "tags": ["тег1", "тег2", "тег3"],
      "featured": true,
      "image": "URL изображения (опционально)",
      "url": "URL детальной страницы (опционально)",
      "createdAt": "2024-01-01T00:00:00Z",
      "updatedAt": "2024-01-01T00:00:00Z"
    }
  ]
}
```

### 3. Активация API

В файле `CasesPage.jsx` раскомментируйте код загрузки из API:

```javascript
useEffect(() => {
  fetchCases()
    .then(data => {
      setCases(data);
      setLoading(false);
    })
    .catch(() => {
      setCases(casesData); // Fallback на локальные данные
      setLoading(false);
    });
}, []);
```

Или используйте функцию `fetchCases` из `casesData.js`, которая уже подготовлена для работы с API.

### 4. Поля кейса

#### Обязательные поля:
- `id` (number) - Уникальный идентификатор
- `title` (string) - Заголовок кейса
- `client` (string) - Название клиента
- `description` (string) - Описание проекта
- `category` (string) - Категория (используется для фильтрации)
- `period` (string) - Период работы
- `results` (array) - Массив результатов с метриками
- `tags` (array) - Массив тегов
- `featured` (boolean) - Признак рекомендуемого кейса

#### Опциональные поля:
- `image` (string) - URL изображения
- `url` (string) - URL детальной страницы кейса
- `createdAt` (string) - Дата создания (ISO format)
- `updatedAt` (string) - Дата обновления (ISO format)

### 5. Категории

Доступные категории для фильтрации:
- `SEO`
- `GEO / AI SEO`
- `Разработка`
- `Автоматизация`

### 6. Результаты

Поле `results` должно содержать массив объектов с метриками:

```javascript
[
  { label: "Рост трафика", value: "+250%" },
  { label: "Увеличение конверсий", value: "+180%" }
]
```

### 7. Теги

Поле `tags` - массив строк для категоризации и поиска:

```javascript
["SEO", "Техническая оптимизация", "Контент-стратегия"]
```

## Пример использования в админке

При создании/редактировании кейса в админке необходимо убедиться, что все обязательные поля заполнены, а структура данных соответствует указанной выше схеме.

## Миграция

При переходе на API/админку:

1. Обновите функцию `fetchCases` в `casesData.js` для работы с вашим API
2. Раскомментируйте код загрузки в `CasesPage.jsx`
3. Убедитесь, что API возвращает данные в правильном формате
4. Протестируйте загрузку и отображение кейсов
5. Удалите локальные данные из `casesData.js` (или оставьте как fallback)
